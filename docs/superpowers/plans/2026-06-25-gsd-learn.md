# `/gsd-learn` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `/gsd-learn` — an inline skill that teaches the 84 concepts in `learn-catalog.md` through a five-beat pattern (concept → how to implement → when/calibration → why → practice), sourced from the references, personalized by the profile, with an optional in-browser visual layer — runnable standalone with no repo.

**Architecture:** The main agent teaches inline (no subagents) by reading the catalog as an index and the cited reference section on demand. A small `gsd-tools learn` handler owns catalog parsing and the user-global progress state (the only persisted artifact). A vendored superpowers-style local server renders the visual beats; terminal is the always-available fallback. The skill is auto-generated from the command file at install.

**Tech Stack:** Node.js CommonJS (`gsd-tools.cjs` + `src/*.cts` → `gsd-core/bin/lib/*.cjs` via `tsc -p tsconfig.build.json`); `node:test` + `node:assert/strict` tests via `tests/helpers.cjs` (`runGsdTools`); GSD command/workflow/reference markdown; graphviz (`dot`) for diagrams; a vendored Node WebSocket dev server for the browser.

- [ ] **Step 1: Write the doctrine**

Write `gsd-core/references/teaching-pattern.md` with these exact sections and content:

- **Title + purpose** — "Teaching Pattern — How `/gsd:learn` Teaches a Concept". State: every node in `learn-catalog.md` is taught through five beats; each beat's content is *derived from the node's `Source` reference section, never invented*; the arc is **what it is → how to build it → when to build it (and when not) → why → make it yours**.
- **The five beats** (a `##` per beat, with the "draws from" pointer):
  1. **Concept** — what it actually is, defined once and clearly; names and dispels the common confusions (hexagonal = clean = onion; mock ≠ spy ≠ stub ≠ fake ≠ dummy). Draws from the skill's definitions.
  2. **How to implement it** — the concrete construction with the reference's real code (where it lives, how it's wired, the pattern). This is *production*, not API syntax. Draws from the skill's code/patterns.
  3. **When to use what** — the calibration: floor / rungs / triggers / both-direction tells, taught through the **good-vs-bad contrast**. Draws from the skill's decision rules.
  4. **Why** — universal evidence first; *then* the subordinate coherence capstone ("why the framework holds you to it"). A learner who never uses GSD still gets full value. Draws from the skill's basis + gates.
  5. **Practice + check** — a now-you-try on a **constructed scenario** (no repo required); optional bonus: grep the learner's repo for a real instance.
- **Two distinctions the pattern keeps separate** — production vs recognition (beat 2 ≠ beat 3); application vs calibration (not rivals; calibration taught through construction + contrast, never abstract preaching).
- **Personalization** — read `~/.claude/gsd-core/USER-PROFILE.md`: `Learning Style` sets order *within* a beat (example-driven → lead with code; conceptual → lead with the principle; hands-on → more drills; self-directed → map then step back); `Explanation Depth` sets prose vs terse. The **calibration mirror**: track over/under/on-target across drills and reflect the lean back.
- **Cross-concept teaching** — when the learner asks how two concepts relate, load both nodes and teach the connection live (the catalog's prereq edges name the links). Follow arbitrary follow-ups inline.
- **Standalone** — no repo or project required; the practice drill is a constructed scenario.
- **The visual layer** — when a node is flagged `Visual` in the catalog, or the learner is stuck or asks to see it, the agent MAY launch the visual companion (Phase 4) for that beat; terminal is always the fallback.
- **Consumes / produces** footer — consumes `learn-catalog.md` (the index) + the cited reference + `USER-PROFILE.md`; produces nothing persistent except progress (via `gsd-tools learn`).

```typescript
import fs from 'node:fs';
import path from 'node:path';
import os from 'node:os';

export interface CatalogNode {
  id: string;
  name: string;
  track: string;
  source: string;     // "reference.md § Section"
  prereqs: string[];   // node ids
  visual: string;      // 'diagram' | 'code' | '—'
}

// The catalog ships with the framework: gsd-core/references/learn-catalog.md.
// learn.cjs lives at gsd-core/bin/lib/, so the catalog is ../../references/.
function catalogPath(): string {
  return path.join(__dirname, '..', '..', 'references', 'learn-catalog.md');
}

// Parse the per-track markdown tables. Rows look like:
// | `id` | Concept text | `file.md § Section` | `prereq-a`, `prereq-b` | diagram |
function parseCatalog(): { count: number; nodes: CatalogNode[]; tracks: string[] } {
  const text = fs.readFileSync(catalogPath(), 'utf8');
  const lines = text.split('\n');
  const nodes: CatalogNode[] = [];
  const tracks: string[] = [];
  let currentTrack = '';
  for (const line of lines) {
    const trackMatch = line.match(/^##\s+Track\s+\d+\s+[—-]\s+(.+?)\s*$/);
    if (trackMatch) {
      // Strip a trailing "(the build wedge)" style annotation and source-skill note.
      currentTrack = trackMatch[1].replace(/\s*\(.*\)\s*$/, '').trim();
      if (!tracks.includes(currentTrack)) tracks.push(currentTrack);
      continue;
    }
    // Table data row: starts with "| `" and has 5 pipe-delimited cells.
    if (!/^\|\s*`[^`]+`\s*\|/.test(line)) continue;
    const cells = line.split('|').slice(1, -1).map(c => c.trim());
    if (cells.length < 5) continue;
    const id = (cells[0].match(/`([^`]+)`/) || [])[1];
    if (!id) continue;
    const prereqCell = cells[3];
    const prereqs = prereqCell === '—' ? []
      : (prereqCell.match(/`([^`]+)`/g) || []).map(s => s.replace(/`/g, ''));
    nodes.push({
      id,
      name: cells[1],
      track: trackToShort(currentTrack),
      source: cells[2].replace(/`/g, ''),
      prereqs,
      visual: cells[4].replace(/[`*]/g, '').trim(),
    });
  }
  return { count: nodes.length, nodes, tracks };
}

// Short track label used by tests/UX (e.g. "Testing (the build wedge)" -> "Testing").
function trackToShort(track: string): string {
  return track.split('(')[0].trim();
}

function prereqChain(nodes: CatalogNode[], id: string): string[] {
  const byId = new Map(nodes.map(n => [n.id, n]));
  const seen = new Set<string>();
  const out: string[] = [];
  const walk = (nid: string) => {
    const node = byId.get(nid);
    if (!node) return;
    for (const p of node.prereqs) {
      if (seen.has(p)) continue;
      seen.add(p);
      walk(p);
      out.push(p);
    }
  };
  walk(id);
  return out;
}

function cmdCatalog() {
  return parseCatalog();
}

function cmdNode(id: string) {
  const { nodes } = parseCatalog();
  const node = nodes.find(n => n.id === id);
  if (!node) return { error: `unknown node: ${id}` };
  return { ...node, prereq_chain: prereqChain(nodes, id) };
}

export = { parseCatalog, prereqChain, cmdCatalog, cmdNode };
```

- [ ] **Step 3: Implement progress in `src/learn.cts`** (replace the Task-2 stubs)

```typescript
interface ConceptProgress {
  id: string;
  status: 'not-started' | 'in-progress' | 'completed';
  lean?: 'over' | 'under' | 'on-target';
  last_seen: string;
}
interface Progress {
  version: string;
  updated: string;
  calibration_lean: { over: number; under: number; on_target: number };
  concepts: ConceptProgress[];
}

function progressDir(): string {
  return process.env.GSD_LEARN_PROGRESS_DIR
    || path.join(os.homedir(), '.claude', 'gsd-core');
}
function progressPath(): string {
  return path.join(progressDir(), 'LEARNING-PROGRESS.md');
}
function emptyProgress(): Progress {
  return { version: '1.0', updated: '', calibration_lean: { over: 0, under: 0, on_target: 0 }, concepts: [] };
}
function readProgress(): Progress {
  const p = progressPath();
  if (!fs.existsSync(p)) return emptyProgress();
  const text = fs.readFileSync(p, 'utf8');
  const m = text.match(/```json\s*([\s\S]*?)```/);
  if (!m) return emptyProgress();
  try { return { ...emptyProgress(), ...JSON.parse(m[1]) }; }
  catch { return emptyProgress(); }
}
function writeProgress(prog: Progress): void {
  const dir = progressDir();
  if (!fs.existsSync(dir)) fs.mkdirSync(dir, { recursive: true });
  const body = [
    '# Learning Progress',
    '',
    '> Tracked by `/gsd:learn`. The machine state is the JSON block below — edit via the command, not by hand.',
    '',
    '```json',
    JSON.stringify(prog, null, 2),
    '```',
    '',
  ].join('\n');
  fs.writeFileSync(progressPath(), body);
}
function parseFlags(argv: string[]): Record<string, string> {
  const out: Record<string, string> = {};
  for (let i = 0; i < argv.length; i++) {
    if (argv[i].startsWith('--')) { out[argv[i].slice(2)] = argv[i + 1]; i++; }
  }
  return out;
}
function cmdProgressRead(): Progress { return readProgress(); }
function cmdProgressUpdate(argv: string[]): Progress {
  const f = parseFlags(argv);
  if (!f.id) return { ...emptyProgress(), updated: 'error: --id required' } as Progress;
  const prog = readProgress();
  const now = new Date().toISOString();
  let entry = prog.concepts.find(c => c.id === f.id);
  if (!entry) { entry = { id: f.id, status: 'in-progress', last_seen: now }; prog.concepts.push(entry); }
  if (f.status) entry.status = f.status as ConceptProgress['status'];
  entry.last_seen = now;
  if (f.lean === 'over' || f.lean === 'under' || f.lean === 'on-target') {
    entry.lean = f.lean;
    const key = f.lean === 'on-target' ? 'on_target' : f.lean;
    (prog.calibration_lean as Record<string, number>)[key]++;
  }
  prog.updated = now;
  writeProgress(prog);
  return prog;
}
```

function cmdNext() {
  const { nodes } = parseCatalog();
  const prog = readProgress();
  const done = new Set(prog.concepts.filter(c => c.status === 'completed').map(c => c.id));
  const candidate = nodes.find(n => !done.has(n.id) && n.prereqs.every(p => done.has(p)));

  return { ...candidate, prereq_chain: prereqChain(nodes, candidate.id) };

'use strict';

const { test, describe } = require('node:test');
const assert = require('node:assert/strict');
const fs = require('node:fs');
const path = require('node:path');

const ROOT = path.resolve(__dirname, '..');
const REFS = path.join(ROOT, 'gsd-core', 'references');
const learn = require(path.join(ROOT, 'gsd-core', 'bin', 'lib', 'learn.cjs'));

describe('learn-catalog integrity', () => {
  const { nodes } = learn.parseCatalog();
  const ids = new Set(nodes.map(n => n.id));

  test('no duplicate node ids', () => {
    assert.equal(ids.size, nodes.length, 'duplicate id in catalog');
  });

  test('every Source reference file exists', () => {
    for (const n of nodes) {
      const file = n.source.split('§')[0].trim();
      assert.ok(fs.existsSync(path.join(REFS, file)),
        `node ${n.id}: source file ${file} missing`);
    }
  });

  test('every Prereq id resolves to a node', () => {
    for (const n of nodes) {
      for (const p of n.prereqs) {
        assert.ok(ids.has(p), `node ${n.id}: prereq "${p}" is not a catalog node`);
      }
    }
  });

  test('prereq graph is acyclic', () => {
    const byId = new Map(nodes.map(n => [n.id, n]));
    const WHITE = 0, GRAY = 1, BLACK = 2;
    const color = new Map(nodes.map(n => [n.id, WHITE]));
    const visit = (id, stack) => {
      color.set(id, GRAY);
      for (const p of (byId.get(id)?.prereqs || [])) {
        if (color.get(p) === GRAY) {
          assert.fail(`cycle: ${[...stack, id, p].join(' -> ')}`);
        }
        if (color.get(p) === WHITE) visit(p, [...stack, id]);
      }
      color.set(id, BLACK);
    };
    for (const n of nodes) if (color.get(n.id) === WHITE) visit(n.id, []);
  });
});
```

#!/usr/bin/env bash
# Render a graphviz DOT string (stdin) to an SVG file (arg 1). Falls back gracefully.

OUT="${1:?usage: render-diagram.sh <out.svg>}"
if ! command -v dot >/dev/null 2>&1; then
